# SWARMSY Workflow Playbooks

## 1) New User Onboarding Workflow
- Trigger: First interaction in workspace
- Required inputs: Identity mode choice, intake responses
- SPARKY steps: Introduce role -> choose mode -> run intake -> set initial locks -> produce first daily command
- Output: Onboarding summary + task starter pack
- Quality gate: Identity consistency + no fake claims
- Next action: Start Identity Compiler

## 2) Hidden Identity Workflow
- Trigger: User chooses anonymous/mask path
- Required inputs: Alias, privacy boundaries, goals
- SPARKY steps: Set Privacy/Identity locks -> craft alias narrative -> define safe exposure strategy
- Output: Hidden identity operating brief
- Quality gate: Privacy lock + legal-safe activation
- Next action: Build lore and signal plan

## 3) Face Identity Workflow
- Trigger: User chooses public-face path
- Required inputs: Public persona details, proof baseline
- SPARKY steps: Lock public identity -> map proof-safe positioning -> define first visibility campaign
- Output: Face identity launch brief
- Quality gate: Proof gate + identity consistency
- Next action: Produce Day 1 campaign assets

## 4) Day 1 Campaign Workflow
- Trigger: Campaign launch day
- Required inputs: Campaign plan, assets, channels
- SPARKY steps: Prioritize one signal -> create AI drafts -> split human execution -> define proof capture
- Output: Day 1 execution pack
- Quality gate: Actionability + no spam + proof
- Next action: Run Day 2 continuation

## 5) Day 2 Continuation Workflow
- Trigger: Day 1 completed
- Required inputs: Day 1 results, proof logs
- SPARKY steps: Review outcomes -> adapt messaging -> continue campaign chapter
- Output: Day 2 continuation plan
- Quality gate: Proof + user workload sanity
- Next action: Continue daily loop

## 6) 30-Day Campaign Workflow
- Trigger: User requests full campaign system
- Required inputs: Goals, channels, product ladder, availability
- SPARKY steps: Build campaign calendar -> assign daily signals -> map weekly checkpoints
- Output: 30-day momentum map
- Quality gate: Measurable + commercial usefulness
- Next action: Start Day 1 workflow

## 7) Product Drop Workflow
- Trigger: New offer or product release
- Required inputs: Product details, audience, launch date
- SPARKY steps: Position offer -> create drop narrative -> build launch tasks and proof metrics
- Output: Product drop brief and checklist
- Quality gate: Commercial usefulness + proof
- Next action: Trigger PR push workflow

## 8) PR Push Workflow
- Trigger: Need media/outreach amplification
- Required inputs: Story angle, proof artifacts, contacts/channels
- SPARKY steps: Draft press assets -> segment outreach -> schedule sends
- Output: PR outreach pack
- Quality gate: No fake claims + research-needed
- Next action: Track responses in proof tracker

## 9) Legal-Safe Activation Workflow
- Trigger: User wants high-visibility activation
- Required inputs: Location/context, constraints, approvals
- SPARKY steps: Check legal boundaries -> propose compliant activation -> prepare risk checklist
- Output: Legal-safe activation plan
- Quality gate: Legal-safe activation gate
- Next action: Execute with documented proof

## 10) Weekly Review Workflow
- Trigger: Weekly checkpoint reached
- Required inputs: Task board, campaign logs, proof tracker
- SPARKY steps: Summarize wins/losses -> audit proof -> reset priorities
- Output: Weekly review report
- Quality gate: Honest + measurable + actionable
- Next action: Generate next-week daily command priorities

## 11) Stuck User Recovery Workflow
- Trigger: User reports overwhelm or no progress
- Required inputs: Current blockers, available time, current assets
- SPARKY steps: Reduce scope -> choose one high-impact move -> rebuild momentum stack
- Output: 24-hour recovery plan
- Quality gate: User workload sanity + actionability
- Next action: Complete one proof-producing action today

## 12) Missing Asset Recovery Workflow
- Trigger: Required assets unavailable
- Required inputs: Missing asset list, substitutes, deadlines
- SPARKY steps: Flag missing assets -> request exact uploads -> offer temporary fallback execution
- Output: Asset recovery checklist
- Quality gate: Actionability + no fake proof
- Next action: Move blocked tasks to ready when assets arrive

## 13) Weak Local Model Recovery Workflow
- Trigger: Local model output quality is poor
- Required inputs: Model name, task type, quality issue
- SPARKY steps: Simplify prompt scope -> chunk tasks -> tighten output format -> mark review points
- Output: Reduced-complexity execution plan
- Quality gate: Useful + honest + specific
- Next action: Human review then continue campaign loop
