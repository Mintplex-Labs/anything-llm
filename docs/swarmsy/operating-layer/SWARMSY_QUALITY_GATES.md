# SWARMSY Quality Gates

## Gate Set

- No fake claims gate
- No spam gate
- Legal-safe activation gate
- Proof gate
- Identity consistency gate
- Lore consistency gate
- Commercial usefulness gate
- Actionability gate
- Research-needed gate
- User workload sanity gate

## Universal Output Checklist

Each output must be checked against:

- Is it specific?
- Is it useful?
- Is it honest?
- Is it legal-safe?
- Is it on-brand?
- Is it measurable?
- Does it create a next action?
- Does it avoid fake proof?

## Gate Definitions

| Gate | Pass condition | Fail response |
|---|---|---|
| No fake claims gate | All claims are verified or clearly labeled pending | Remove claim, add proof task |
| No spam gate | Distribution plan is platform-safe and non-deceptive | Rewrite as legal signal plan |
| Legal-safe activation gate | Actions avoid illegal/reckless tactics | Replace with compliant alternatives |
| Proof gate | Claim-to-proof map exists and is current | Mark proof gap and block finalization |
| Identity consistency gate | Output aligns with Identity/Voice locks | Revise output to lock state |
| Lore consistency gate | Output does not violate lore canon | Correct against lore lock |
| Commercial usefulness gate | Output advances offers, reach, or conversion path | Re-scope to business impact |
| Actionability gate | Output contains concrete next step(s) | Add specific actions and owners |
| Research-needed gate | Time-sensitive claims verified or flagged | Add verification requirement |
| User workload sanity gate | Plan fits realistic user capacity | Reduce scope and prioritize |
