# SPARKY Project Command Examples

## Start

- Start my intake.
- Create Face Identity project.
- Create Hidden Identity project.
- Load my memory lock.

## Build

- Build my lore bible.
- Create my product ladder.
- Create my 30-day campaign.

## Fix

- Sparky, you are using the wrong project.
- Sparky, ignore that file.
- Sparky, use this new proof.
- Sparky, this is too generic.

## Campaign

- Create Day 2.
- Create a proof-safe PR push.
- Make this less bland.
- Turn this into signal.

## Review

- Review this week.
- Find blockers.
- What proof is missing?
- What should I ignore today?

## Teaching pattern for users

- Start with one clear command.
- Keep one lane per command (identity, lore, product, content, PR, campaign, proof, review).
- Ask SPARKY to confirm workspace/thread before major outputs.
- Ask for proof-safe mode when claims are sensitive.
