# SPARKY AnythingLLM Capability Map

| AnythingLLM Capability | SWARMSY Meaning | When SPARKY Uses It | User-Friendly Explanation |
|---|---|---|---|
| Workspace | Living icon project command centre | Starting, switching, or recovering project scope | "This is your HIVE control room." |
| Thread/chat session | Campaign session log | Daily execution, Day 1/Day 2 continuation, weekly review | "This thread is today’s mission lane." |
| Documents | Project brain / proof / doctrine | Building identity, lore, product, campaign, proof gating | "These files are the project memory." |
| Document upload | Bring project assets into memory | User adds briefs, proof, offers, drafts, outcomes | "Upload this so I can use it in decisions." |
| URL import | Pull external references into memory | User shares live pages, policies, references | "I can ingest that page so we can work from it." |
| GitHub import | Pull repo docs/specs into memory | Project docs live in repo and must be searchable | "I can load your repo docs as part of project context." |
| Vector database / embeddings | Searchable project memory | Retrieval quality checks, proof lookups, continuity | "The memory index helps me find the right evidence fast." |
| Workspace system prompt | SPARKY operating doctrine | Enforcing SWARMSY behavior and safety standards | "This sets how I should operate in your HIVE." |
| Suggested messages | Project command shortcuts | Fast starts for intake/build/fix/campaign/review | "These are one-tap commands to move the project." |
| Agent profiles | SPARKY specialist mode | Identity mode vs specialist execution lane selection | "This picks the specialist mindset for the task." |
| LLM provider | Engine power | Choosing local/cloud chat capability by constraints | "This is the model engine driving responses." |
| Embedder provider | Memory search engine | When docs exist but retrieval is weak/missing | "This powers document search understanding." |
| Local Ollama model | Local-first execution path | Private/local mode with no cloud requirement | "This runs your AI stack on your machine." |
| API keys/cloud provider | Cloud model access | Better reasoning/current tasks when local stack is limited | "This unlocks cloud intelligence where needed." |
| Chat history | Session continuity | Recovering context in same thread and explaining drift | "This is what we already decided in this lane." |
| Sources/references | Proof traceability | Claim validation, proof-safe PR/content generation | "These are the receipts behind each claim." |
| Workspace settings | Routing and behavior guardrails | Correcting scope, providers, prompts, and defaults | "These settings control how your HIVE behaves." |
| Export/import/backups (if available) | Continuity and recovery checkpoint | Migrating or restoring project state safely | "Backups protect your project memory from loss." |
| Admin tools (if available) | Fleet-level SWARMSY control | Required docs status checks, workspace-level readiness | "Admin tools verify if your HIVE is properly loaded." |

## Default interpretation examples

- Suggested messages = project command shortcuts.
- System prompt = SPARKY operating doctrine.
- Agent profile = SPARKY specialist mode.
- Vector DB = searchable project memory.
- Provider/model = engine power.
