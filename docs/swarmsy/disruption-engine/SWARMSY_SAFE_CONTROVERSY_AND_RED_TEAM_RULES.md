# SWARMSY Safe Controversy and Red-Team Rules

## Principle

Controversy is not the goal. Trustworthy signal is the goal.

Attention without trust is useless.

## Hard “Never” Rules

- Never fake outrage
- Never target protected groups
- Never harass
- Never impersonate authorities
- Never create fake emergencies
- Never invent enemies
- Never lie for PR
- Never use tragedy for cheap attention
- Never create doxxing or brigading

## Safer Controversy Types

- challenging blandness
- calling out fake hype
- questioning algorithm culture
- anti-corporate sameness
- anti-disposable content
- pro-human identity
- local pride
- anti-invisibility
- artist survival

## Execution Rules

1. Critique systems/patterns, not vulnerable people.
2. Use verifiable statements and explicit context.
3. Offer constructive alternative behavior.
4. Keep tone sharp but non-abusive.
5. Escalate only with legal-safe fallback options.

## Red-Team Checklist

- Could this be misunderstood as a threat?
- Could this look like harassment?
- Could this create legal trouble?
- Could this damage trust?
- Could this rely on a lie?
- Could this attract the wrong audience?
- Could this hurt a real person/group?
- Could this be made sharper without that risk?

## Decision Gate

- If any checklist answer is “yes,” revise or reject.
- If risk remains unresolved, default to safer controversy type.
- Document risk, mitigation, and fallback before publication.

## Integrity Standards

- No fake claims, bots, engagement, press, sales, or community
- No spam, platform manipulation, harassment, or illegal tactics
- Replace spam with signal
- Break the mould, not the law
- Shock the attention system, not the law
