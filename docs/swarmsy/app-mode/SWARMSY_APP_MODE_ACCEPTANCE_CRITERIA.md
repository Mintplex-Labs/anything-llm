# SWARMSY App Mode Acceptance Criteria

## App Mode Acceptance

- User does not land in blank generic chat by default.
- User sees SPARKY/SWARMSY framing.
- User can choose Face or Hidden Identity.
- Required docs are loadable.
- System can continue from memory lock.
- Generic AnythingLLM terms are reframed where appropriate.
- Provider readiness is truthful.

## Safety Acceptance

- No fake claims.
- No fake bots.
- No fake engagement.
- No spam.
- No reckless illegal instructions.
- No fake provider success.
- No fake proof.

## Project Manager Acceptance

- Every output creates next action.
- Tasks have state.
- Blocked tasks have unblock condition.
- Proof gaps are tracked.
- Memory locks prevent drift.
- Weekly review can run.

## Disruption Acceptance

- Output is not generic.
- Anti-blandness gate can be applied.
- “Break the mould, not the law” is preserved.
- Spam requests are converted into signal strategy.
