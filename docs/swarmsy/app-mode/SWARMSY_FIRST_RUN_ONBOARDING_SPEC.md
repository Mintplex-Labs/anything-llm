# SWARMSY First-Run Onboarding Spec

## First Screen Copy

Welcome to SWARMSY HIVE.

Are we building you as the face, or building a hidden identity?

## Primary Buttons

Face Identity Mode
Hidden Identity Mode
Existing Project
Load Memory Lock
Explore Demo

## Button Definitions

### Face Identity Mode

- User is visible.
- Personal story can be used.
- Face/camera/content strategy matters.
- Privacy boundaries still required.

### Hidden Identity Mode

- Alias/persona/mask/symbol identity.
- Privacy-first.
- Trust built through proof, consistency, product, lore, and output.

### Existing Project

- Asks for existing docs/assets/links.
- Creates current state summary.
- Asks what is already real.

### Load Memory Lock

- Lets returning users continue.
- No rebrand unless requested.

### Explore Demo

- Loads a fake/demo project.
- Clearly marked demo.
- No fake real-world claims.

## Provider Check

- Check local provider/model status.
- Recommend Ollama if local AI is missing.
- Do not fake readiness.
- Mark missing model/provider clearly.
